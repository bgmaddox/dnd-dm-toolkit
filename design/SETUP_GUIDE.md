# DnD DM Toolkit — Setup Guide

This is a local DM toolkit that uses Claude AI (via the Anthropic API) to power an NPC Forge and a Combat Companion. You run it on your own machine and access it in a browser. Each person needs their own Anthropic API key.

---

## What You Need

- A computer running macOS, Windows, or Linux
- Python 3.10 or newer ([download here](https://www.python.org/downloads/))
- A free Anthropic account with an API key ([sign up here](https://console.anthropic.com/))
- The project files (zip from me, or cloned from GitHub)

---

## Step 1 — Get the Files

If I sent you a zip, unzip it somewhere convenient (e.g. `~/Documents/dnd-toolkit`).

If it's on GitHub:
```bash
git clone https://github.com/YOUR_REPO_URL dnd-toolkit
cd dnd-toolkit
```

---

## Step 2 — Get an Anthropic API Key

1. Go to [console.anthropic.com](https://console.anthropic.com/) and create an account
2. Go to **API Keys** and click **Create Key**
3. Copy the key — you won't be able to see it again after closing the dialog

The tools use `claude-haiku` which is the cheapest Claude model. Typical usage (generating NPC descriptions, monster tactics) costs fractions of a cent per request. Anthropic gives new accounts some free credits to start.

---

## Step 3 — Create Your `.env` File

In the project folder, create a file named `.env` (no other extension — just `.env`):

```
ANTHROPIC_API_KEY=sk-ant-your-key-here
```

Replace `sk-ant-your-key-here` with the key you copied. This file is never shared or committed to git — it stays on your machine only.

---

## Step 4 — Set Up Python Environment

Open a terminal in the project folder and run:

```bash
# Create a virtual environment
python3 -m venv .venv

# Activate it
# macOS/Linux:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
```

You only need to do this once. After that, just activate the venv each time before running the server.

---

## Step 5 — Run the Server

With the venv active:

```bash
uvicorn server:app --port 8502
```

Wait a moment, then open your browser and go to:

```
http://localhost:8502/tools/npc_forge.html
```

The Combat Companion is at:

```
http://localhost:8502/tools/combat_companion.html
```

To stop the server, press `Ctrl+C` in the terminal.

> **Note:** You need to re-run this command each time you want to use the tools. If you want it to start automatically when your computer boots, that's possible but requires extra setup — ask Claude Code to help you set up a startup service for your OS.

---

## Step 6 — Set Up Your Campaign (Optional)

The `campaign/` folder holds your world knowledge:

```
campaign/
  YOUR_CAMPAIGN_NAME/
    npcs/        ← NPC markdown files
    locations/   ← Location notes
    pcs/         ← Player character sheets (generated by fetch_character.py)
    sessions/    ← Session logs
```

Copy the folder structure and rename `LMoP` to your campaign name. Templates for each file type are in `campaign/templates/`.

### Importing D&D Beyond Characters

If your players use D&D Beyond, you can generate PC markdown files automatically:

1. Log into D&D Beyond in your browser
2. Open the character sheet
3. Open DevTools (F12) → Network tab → filter by `character/v5`
4. Refresh the page, then click the request ending in `?includeCustomItems=true`
5. Go to the **Response** tab → click **Raw Data** → select all → copy
6. Save to a file, e.g. `campaign/characters/thorin.json`
7. Run: `python fetch_character.py --file campaign/characters/thorin.json`

The script outputs a markdown file to `campaign/YOUR_CAMPAIGN/pcs/`.

## Troubleshooting

**Port already in use:** Another app is on port 8502. Change the port in your start command: `uvicorn server:app --port 8503`, then access the tools at `localhost:8503` instead.

**`ANTHROPIC_API_KEY` not set error:** Make sure your `.env` file is in the project root folder (same level as `server.py`) and starts with `ANTHROPIC_API_KEY=`.

**Monsters/spells not loading in Combat Companion:** The `tools/` folder needs to contain `monsters.json`, `spells.json`, `conditions.json`, and `magic_items.json`. These are included in the project files — make sure they weren't left out of the zip.

**Python not found:** Make sure Python 3.10+ is installed and on your PATH. Try `python3 --version` in a terminal.
